# Database Setup

This directory contains the database schema, migrations, and configuration for the Better Auth integration.

## Files

- `schema.js` - Drizzle ORM schema definitions for Better Auth tables
- `index.js` - Database connection and Drizzle instance
- `migrate.js` - Migration runner script
- `migrations/` - SQL migration files generated by Drizzle Kit

## Tables

The database includes the following tables for Better Auth:

1. **user** - Stores user account information
   - id, email, emailVerified, name, createdAt, updatedAt

2. **session** - Stores user session data
   - id, userId, expiresAt, token, ipAddress, userAgent, createdAt, updatedAt

3. **account** - Stores authentication provider information
   - id, userId, accountId, providerId, password, accessToken, refreshToken, etc.

4. **verification** - Stores verification tokens for email verification
   - id, identifier, value, expiresAt, createdAt, updatedAt

## Usage

### Generate new migrations

```bash
npm run db:generate
```

### Run migrations

```bash
npm run db:migrate
```

## Database Location

The SQLite database file is located at `./db/app.db` and is excluded from version control via `.gitignore`.
